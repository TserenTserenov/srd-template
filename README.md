# SRD Template

Шаблон хранилища знаний по методу **SRD (Systems Role Description)** — организация проектной информации через системы и роли.

## Метод SRD

SRD структурирует знания о проекте в виде матрицы **3 системы × 3 роли = 9 ячеек**:

### Три системы

1. **Надсистема** — внешний контекст, в котором существует проект
2. **Целевая система** — сам проект, продукт или система
3. **Система создания** — команда, процессы и инструменты разработки

### Три роли

1. **Смыслы** (Meaning) — зачем, ценности, цели
2. **Архитектура** (Architecture) — что, структура, модели
3. **Операции** (Operations) — как, процессы, действия

### Матрица 3×3

|                    | Смыслы (x.1)       | Архитектура (x.2)     | Операции (x.3)        |
|--------------------|--------------------|-----------------------|-----------------------|
| **1. Надсистема**  | 1.1. Зачем нужна   | 1.2. Что есть вокруг  | 1.3. Как работает мир |
| **2. Целевая**     | 2.1. Зачем проект  | 2.2. Что в проекте    | 2.3. Как работает     |
| **3. Создания**    | 3.1. Зачем команде | 3.2. Что в команде    | 3.3. Как разработка   |

### Дополнительно: Управление (0.x)

Отдельная категория для метаинформации о самом хранилище:

- **0.1. Логика хранилища** — правила организации и соглашения

## Структура

- `content/` — основное хранилище документов по системам и ролям
- `.fpf/` — спецификации First Principles Framework
- `CLAUDE.md` — инструкции для работы с AI-ассистентом

## Использование

1. Размещайте документы в соответствующие папки по системам и ролям
2. Используйте шаблоны документов с YAML frontmatter
3. Следуйте принципам минимализма и практичности
